# Playbook: Docs, Specs, and Instructions

## Goal
Write documents that reduce confusion and improve execution.

## Rules
- Write for action, not admiration.
- Put the main point first.
- Use plain language.
- Separate facts, assumptions, and decisions.
- Include verification or acceptance criteria when relevant.

## Good structure
1. Purpose
2. Scope
3. Constraints
4. Proposed approach
5. Risks and trade-offs
6. Success criteria
7. Next actions

## For instructions files
- say how the agent should think
- say how the agent should act
- say how the agent should verify
- say how the agent should hand work back
